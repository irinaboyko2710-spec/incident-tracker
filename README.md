# Трекер інцидентів кібербезпеки (Лабораторна робота №4) #

## Виконання вимог на оцінку "Відмінно"

### 1. Контракти та версійність
* **Версійність:** Усі API-ендпоінти використовують префікс `/api/v1/` для забезпечення стабільності контрактів (наприклад, `/api/v1/incidents`).
* **Правила сумісності DTO:**
  1. **Адитивність:** Додано нове поле `user_name` (Заявник), яке є необов'язковим для забезпечення зворотної сумісності.
  2. **Незмінність:** Базові поля (`tag`, `severity`, `comments`) зберегли свої назви та типи, щоб не зламати роботу існуючих клієнтів.
  3. **Дефолтні значення:** Реалізовано механізм підстановки значення "Не вказано", якщо поле заявника прийшло від клієнта порожнім.

### 2. Технічна реалізація
* **Frontend:** Побудований на **TypeScript**. Використано інтерфейси для типізації об'єктів інцидентів, що виключає помилки невідповідності даних.
* **Backend:** Тришарова архітектура (Controller -> Service -> Repository). Дані надійно зберігаються в реляційній базі даних **SQLite**.
* **Надійність:** Реалізовано "безпечний рендеринг" у `ui.ts` (обробка `null` значень дати через тернарний оператор) та `AbortController` для контролю таймаутів запитів.
* **Обробка помилок:** Система коректно відображає повідомлення про помилки при відсутності зв'язку з сервером або проблемах із доступом до БД.

### 3. Сценарії перевірки (QA)
1. **Успішне додавання:** Заповнення форми (включаючи поле "Заявник"). Після натискання "Зберегти" запис з'являється в таблиці з унікальним ID, згенерованим базою даних.
2. **Збереження стану:** Після повного перезавантаження сторінки дані не зникають, оскільки вони записані у фізичний файл `app.db`.
3. **Видалення:** Функція видалення автоматично оновлює інтерфейс та видаляє відповідний рядок із таблиці бази даних.
4. **Обробка відсутності даних:** Якщо при створенні не вказати дату, система замінить її на прочерк ("—"), не перериваючи виконання скриптів і не блокуючи інтерфейс.

## Як запустити
1. **Компіляція TS:** Перейдіть у папку `frontend` та виконайте команду `npx tsc`.
2. **Запуск Backend:** Перейдіть у папку `backend` та запустіть сервер командою `node src/index.js` (сервер працює на порту 3000).
3. **Запуск Frontend:** Відкрийте файл `frontend/index.html` через Live Server (зазвичай порт 5500 або 5501).
