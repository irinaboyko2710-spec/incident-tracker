# Lab 0.3.0 - Трекер інцидентів кібербезпеки (SQLite Edition)

## Опис проекту
Ця версія застосунку використовує реляційну базу даних **SQLite** для надійного зберігання інформації про інциденти та користувачів. Реалізовано автоматичну ініціалізацію структури БД при першому запуску.

## Як запустити
1. Переконайтеся, що встановлено **Node.js**.
2. Виконайте команду `npm install` для встановлення залежностей.
3. Запустіть сервер командою `node src/index.js`. 
4. **Важливо:** При першому старті програма автоматично створить папку `data`, файл `app.db` та ініціалізує всі необхідні таблиці.

## Опис схеми бази даних
База даних містить дві пов'язані таблиці:

### 1. Таблиця `users`
Зберігає інформацію про користувачів, які реєструють інциденти.
* `id`: INTEGER PRIMARY KEY (автоінкремент).
* `name`: Текстове ім'я користувача.
* `email`: Електронна пошта (унікальна).

### 2. Таблиця `incidents`
Зберігає журнал подій безпеки.
* `id`: INTEGER PRIMARY KEY.
* `date`: Дата та час (за замовчуванням `CURRENT_TIMESTAMP`).
* `tag`: Категорія інциденту (**Access, Malware, Network, Data, DDoS**).
* `severity`: Рівень важливості (**Low, Medium, High**).
* `comments`: Детальний опис.
* `user_id`: FOREIGN KEY, що посилається на `users(id)`.
